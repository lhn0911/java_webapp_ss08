Quy tắc đặt tên

Quy tắc đặt tên là tập hợp các nguyên tắc giúp đảm bảo các endpoint, tài nguyên, và tham số được đặt tên một cách nhất quán, dễ hiểu, và tuân theo chuẩn REST. Một Naming Convention tốt giúp:

  • Tăng tính đọc hiểu (readability) cho các nhà phát triển.
  • Đảm bảo tính nhất quán trong thiết kế API.
  • Dễ dàng bảo trì và mở rộng API.

1. Tài nguyên chính

Tên tiếng Việt Tên tài nguyên (resource) Endpoint gốc (base path)
Sách Book /books
Tác giả Author /authors
Thể loại Category /categories

2. Các phương thức HTTP & Endpoint mẫu

2.1. Sách (/books)

Hành động Method Endpoint Mô tả
Lấy danh sách sách GET /books Trả về danh sách tất cả sách
Lấy sách theo ID GET /books/{id} Trả về thông tin sách cụ thể
Thêm sách mới POST /books Tạo sách mới
Cập nhật sách PUT /books/{id} Cập nhật toàn bộ sách
Xoá sách DELETE /books/{id} Xoá sách theo ID

2.2. Tác giả (/authors)

Hành động Method Endpoint Mô tả
Lấy danh sách tác giả GET /authors Trả về danh sách tác giả
Lấy tác giả theo ID GET /authors/{id} Trả về thông tin tác giả
Thêm tác giả mới POST /authors Tạo tác giả mới
Cập nhật tác giả PUT /authors/{id} Cập nhật thông tin tác giả
Xoá tác giả DELETE /authors/{id} Xoá tác giả theo ID

2.3. Thể loại (/categories)

Hành động Method Endpoint Mô tả
Lấy danh sách thể loại GET /categories Trả về danh sách thể loại
Lấy thể loại theo ID GET /categories/{id} Trả về thông tin thể loại
Thêm thể loại mới POST /categories Tạo thể loại mới
Cập nhật thể loại PUT /categories/{id} Cập nhật thông tin thể loại
Xoá thể loại DELETE /categories/{id} Xoá thể loại theo ID

3. Nguyên tắc bổ sung

  • Không dùng động từ trong endpoint: ❌ /getAllBooks → ✅ /books
  • Dùng danh từ số nhiều: /books, /authors, /categories
  • Dùng path parameters cho tài nguyên con: /books/{id}/authors
  • Dùng query parameters để lọc/tìm kiếm: /books?title=java&category=backend
